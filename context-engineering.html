Context Engineering

A prompt-along experience. Build context, watch your output transform.

Think about the best manager you've ever had. When you started a new project, they didn't just say "build the thing" and walk away. They onboarded you — shared the vision, explained who the audience was, pointed you to the brand guidelines, walked you through what success looks like.

Working with AI is the same job. Claude is remarkably capable, but it's new to your project every time. Your role is to be the manager who onboards it well — and the better that onboarding, the better the work.

This is a prompt-along. We'll walk through all the different ways you can onboard Claude to your project — from a simple chat message all the way to a full documentation system. You'll type real prompts, build real files, and watch the output transform as the onboarding gets richer. No coding experience needed.

You're the manager. Claude is your new hire.

AI doesn't follow instructions like a calculator. It interprets your context and composes solutions — like a talented new team member would. The question isn't "how do I write a better prompt?" It's "how do I set this person up to succeed?"

Step 1 — Create your project folder
  1. Go to your Desktop (minimize any open windows so you can see it)
  2. Right-click on an empty spot on your Desktop
  3. Select "New" → "Folder" (Windows) or "New Folder" (Mac)
  4. Name it context-engineering
Step 2 — Open Claude and connect to your folder
  1. Open the Claude desktop app on your computer
  2. Look for the "Code" toggle at the bottom of the chat input — click it to turn it on. This activates Claude Code, which can read and write real files on your computer.
  3. You'll see a prompt to select a project folder. Click it, then navigate to your Desktop and select the context-engineering folder you just created.
  4. You're connected. Every file Claude creates will appear inside that folder on your Desktop.
What is Claude Code?
Claude Code is a mode inside the Claude desktop app that lets Claude read and write real files on your computer. When you turn it on and connect it to a folder, Claude can create HTML pages, documents, and code — all saved as real files you can open, edit, and share. Think of it as hiring a capable builder and giving them a workspace. The folder is their workspace. Your job is to manage what they build.

The limitation of verbal briefings
A verbal briefing works for one task. But what happens when you close this conversation and start a new one? Claude forgets everything. You'd have to re-brief every time. That's like onboarding the same employee from scratch every morning. Fortunately, the Claude desktop app gives you other levers — ways to create onboarding that persists, so Claude shows up prepared every time.

When you onboard a new team member, you don't just brief them verbally — you give them written documentation. A handbook, a wiki, a "start here" document.

Claude has exactly this: a file called CLAUDE.md. When you start a new conversation in the Claude desktop app with Code mode on, it automatically reads this file first — before you even type anything. It's your project's onboarding document. Write it once, and Claude arrives prepared every time.

Prompt #3 — Paste this into Claude
Create a CLAUDE.md file in the root of this project with the following project instructions: # Context Engineering Project ## Brand: Morning Light Coffee - Quirky neighborhood coffee shop, corner of Oak & Fifth - Roasts own beans, hosts community events, names drinks creatively - Vibe: part coffee shop, part living room, part community center - Tagline: "Where every morning is a good morning" - Never use: "fast service," "grab and go," "premium blend," "artisan crafted" ## Visual Identity - Primary: deep forest green #2d4a3e - Accent: terracotta #c4704b - Highlight: warm gold #d4a853 - Background: cream #fdf6ec - Chalkboard: near-black #1e1e1e (for menu sections) - Headlines: Playfair Display (regular weight only, NEVER bold) - Body text: Inter (regular 400, semi-bold 600) - Never use pure black text or cold gray backgrounds ## Rules - All output should be self-contained HTML files (no frameworks) - Design should feel handcrafted and warm, like the shop itself - Always include generous whitespace and rounded corners - Chalkboard sections use dark background with warm-toned text

Claude just created a file called CLAUDE.md in your project folder. If you open your context-engineering folder on your Desktop, you'll see it there alongside index.html. From now on, every new conversation you start with Claude in this folder will read that file first and follow those rules.

This is the full managerial loop: you direct, Claude builds, you review, and the documentation evolves. You don't need to read every document yourself — that's Claude's job. Your job is to manage the inputs, evaluate the outputs, and make sure decisions get recorded. Every correction becomes a permanent improvement to how Claude understands your project.
